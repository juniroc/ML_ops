# Too many pipes pattern

- 학습 파이프라인이 너무 다양하고 복잡한 경우
- 최대한 단순화하여 장애를 격리하고 재사용성을 높이는 것이 중요
- 데이터가 여러 DWH에 저장될 경우
→ 액세스, 인증 및 권한 부여를 정의하는 작업이 증가
→ 적절한 DWH 를 선택하고 정리하는 것이 중요

**장점**

- 복잡한 파이프라인을 통해 다양한 작업 실행

**단점**

- 운영이 어려움
